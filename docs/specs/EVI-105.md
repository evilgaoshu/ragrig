# EVI-105: Audit Trail & Resume Guardrails

## Objective

Extend the source save / dry-run / retry-resume operations delivered in
[EVI-102](./EVI-102.md) with:

1. **Audit event recording** for every mutating workflow operation
2. **Guardrails** that prevent unsafe or duplicate retry/resume
3. **Console audit summary** display showing the event trail

## Audit Event Types

| Event Type          | Trigger                        | Actor Field | Payload Summary                            |
|---------------------|--------------------------------|-------------|--------------------------------------------|
| `source_save`       | Source config saved            | operator    | source_id, kind, uri                       |
| `dry_run_start`     | Dry-run scan initiated         | —           | plugin_id                                  |
| `dry_run_complete`  | Dry-run scan finished          | —           | plugin_id, total, discovered/skipped/failed counts |
| `retry_start`       | Retry of an item/run started   | operator    | document_uri, run_type                     |
| `retry_complete`    | Retry finished                 | operator    | status (success/failed), error_message, new_version_number |
| `resume_start`      | Resume of a run started        | operator    | run_id, run_type                           |
| `resume_complete`   | Resume finished                | operator    | status, retried/succeeded/failed counts    |

## Database Schema

All events are stored in the `audit_events` table with the following columns:

| Column            | Type      | Notes                                    |
|-------------------|-----------|------------------------------------------|
| id                | UUID PK   | Auto-generated                           |
| event_type        | String(64)| Indexed — one of the event types above   |
| actor             | String(255)| nullable — operator identifier           |
| knowledge_base_id | UUID FK   | nullable, SET NULL on delete             |
| document_id       | UUID FK   | nullable, SET NULL on delete             |
| chunk_id          | UUID FK   | nullable, SET NULL on delete             |
| run_id            | UUID FK   | nullable, SET NULL on delete — pipeline_runs.id |
| item_id           | UUID FK   | nullable, SET NULL on delete — pipeline_run_items.id |
| occurred_at       | DateTime  | server default now()                      |
| payload_json      | JSONB     | sanitized via `_safe_payload`            |

## Guardrail Rules

### 1. Invalid State Transition
- **Condition**: `item.status != "failed"`
- **Response**: `{ "denied": true, "reason": "invalid_state_transition", ... }`
- **HTTP**: 200 with denied body (not an HTTP error)

### 2. Duplicate Retry (already succeeded)
- **Condition**: `item.metadata_json.retry_version_number is not null AND item.status == "success"`
- **Response**: `{ "denied": true, "reason": "duplicate_retry", ... }`

### 3. Concurrent Retry
- **Condition**: `item.metadata_json.retry_in_progress == true`
- **Response**: `{ "denied": true, "reason": "concurrent_retry", ... }`

### 4. Expired Snapshot (item-level)
- **Condition**: `run.config_snapshot_json.snapshot_expired == true` OR `snapshot_age > 86400s` (24h)
- **Response**: `{ "denied": true, "reason": "expired_snapshot", ... }`

### 5. Expired Snapshot (run-level)
- Same logic as item-level, applied to the run before retrying all failed items.

## Secret Boundary

All audit output (API responses, Console display) MUST NOT contain:

- Raw `password`, `api_key`, `token`, `secret`, `raw_secret`, `private_key`, `access_key`, `session_token`
- Full original text of configuration values > 240 characters

The `_safe_payload` mechanism (key-name pattern matching + truncation) is applied to
all `payload_json` when writing audit events, and `_redact_summary` is applied when
serving audit-events API responses.

## Console UI

The Console displays a **Workflow Audit Events** panel showing:

| Column      | Content                                |
|-------------|----------------------------------------|
| Timestamp   | `occurred_at` ISO-8601                 |
| Event Type  | Color-coded badge per event type       |
| Actor       | Operator name or `—`                   |
| Run ID      | First 8 chars of UUID or `—`          |
| Item ID     | First 8 chars of UUID or `—`          |
| Summary     | Truncated JSON payload                 |

When no events exist or the endpoint is unreachable, a **DEGRADED** badge is shown
instead of the table (not blank).

## API Endpoints

### `GET /audit-events`
Query parameters:
- `limit` (int, default 50)
- `event_type` (optional filter)
- `run_id` (optional filter)
- `item_id` (optional filter)

Response shape:
```json
{
  "entries": [
    {
      "id": "uuid",
      "event_type": "retry_start",
      "actor": "admin",
      "knowledge_base_id": "uuid",
      "run_id": "uuid",
      "item_id": "uuid",
      "occurred_at": "2026-05-13T00:00:00+00:00",
      "payload": { "document_uri": "file:///data/doc.txt", "run_type": "local_ingest" }
    }
  ]
}
```

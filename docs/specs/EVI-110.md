# EVI-110: Pilot Go/No-Go Evidence Pack

## Goal

Freeze the first pilot decision surface into a reproducible repository deliverable.
The evidence pack is a decision index, not a connector expansion project: it names
the fixed corpus, real-source-equivalent connector path, golden questions, command
set, output artifacts, and go/no-go rules.

## Source Choice

The pilot evidence path uses the live SMB, WebDAV, and SFTP fileshare connector smoke
as an explicit equivalent real source. Google Workspace requires credentials outside a
reproducible public repository, while the fileshare route still exercises a live
network connector, fixture seeding, source scanning, ingestion, and connector-level
failure handling.

Command:

```bash
make test-live-fileshare
```

Artifact:

```text
docs/operations/artifacts/fileshare-live-smoke-record.json
```

## Fixed Pilot Corpus

The corpus is versioned under:

```text
tests/fixtures/fileshare_live/
```

`make pilot-evidence-pack` hashes every file and writes the fixed inventory into:

```text
docs/operations/records/EVI-110-pilot-go-no-go-evidence.md
docs/operations/artifacts/pilot-go-no-go-evidence.json
```

The manifest is the canonical evidence index. The Markdown record is tracked; the JSON
artifact follows the repository's existing local-artifact ignore policy.

## Golden Questions

Pilot retrieval quality uses:

```text
tests/fixtures/evaluation_golden.yaml
```

The fixture covers positive hits, nested content, zero-result refusal context, and
edge cases for empty/binary documents. The evidence manifest copies query labels,
tags, and expected documents so reviewers can see exactly what the quality baseline
means without inferring it from test code.

## CI Automation

`.github/workflows/nightly-evidence-smoke.yml` runs the fixed evidence route every
night and on manual dispatch. PRs that change the workflow, smoke orchestrator, or
evidence-pack contract also run the same job. The workflow installs the fileshare
live SDK extra, sets deterministic Docker ports, runs:

```bash
make nightly-evidence-smoke
```

The target executes the seven evidence groups above, always refreshes the evidence
pack, and uploads the command logs plus these summary artifacts:

```text
docs/operations/artifacts/nightly-evidence-smoke.json
docs/operations/artifacts/nightly-evidence-smoke.md
docs/operations/artifacts/pilot-go-no-go-evidence.json
docs/operations/records/EVI-110-pilot-go-no-go-evidence.md
```

## Decision Items

| Requirement | Go condition | No-go signal |
| --- | --- | --- |
| Real-source equivalent | Live fileshare smoke completes with retained evidence. | Preflight or compose failure is unrecorded or waived without an alternate approved source. |
| ACL does not overreach | `make test` keeps ACL regression and permission preview tests passing. | Cross-scope ACL result or missing ACL regression evidence. |
| Citations remain inspectable | Answer diagnostics and answer/web tests keep citation schema and empty-citation refusal behavior. | Citation shape cannot be checked from artifact/tests. |
| Failed run can be inspected/retried | Pipeline DAG artifact plus retry/audit regression tests remain available. | Failed run collapses into opaque error or duplicate retry loses guardrails. |
| Audit trail remains traceable | Workflow audit endpoint tests pass and outputs remain sanitized. | Missing or secret-bearing audit response. |
| Retrieval/answer baseline recorded | Evaluation and benchmark artifacts are emitted. | Benchmark compare fails, or golden questions are not tied to a fixed fixture. |
| Operations smoke summarized | Backup/restore/upgrade write status-bearing artifacts. | Silent success, missing status field, or missing restore/upgrade artifact. |


## Residual Risk And Boundaries

- Google Workspace is not asserted as repo-reproducible for this issue; the fileshare
  live route is the declared equivalent source for pilot decision evidence.
- `answer-live-smoke` may emit `healthy`, `degraded`, or `skip` depending on local
  provider availability. Each is explicit evidence; only silent absence is unacceptable.
- Live fileshare smoke requires Docker and optional fileshare SDKs. If a lab cannot run
  it, its blocked preflight record is evidence of environment readiness failure, not a pass.
- M365, extra connectors, production HA, and a SaaS control plane stay out of scope.

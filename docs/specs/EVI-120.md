# EVI-120 Retrieval Benchmark Guard CI Visibility

Date: 2026-05-14
Issue: EVI-120
Related issues: EVI-117, EVI-116, EVI-111

## Goal

Expose retrieval benchmark compare and integrity preflight failures as a standalone GitHub PR check instead of hiding them inside the `coverage` job.

## CI Shape

GitHub Actions defines a dedicated `benchmark-guard` job named `RAGRig CI / benchmark-guard`.

The job runs:

1. `make retrieval-benchmark-compare`
2. `make retrieval-benchmark-integrity-artifact`
3. `make retrieval-benchmark-integrity-summary`

The compare step captures stdout/stderr to `docs/operations/artifacts/retrieval-benchmark-compare.log` with `pipefail` enabled, so compare failures still fail the job. The job uploads the compare log plus integrity JSON and Markdown summary as the `retrieval-benchmark-guard` artifact.

The job summary includes the compare output and the integrity Markdown summary for quick PR review.

## Coverage Job Boundary

The `coverage` job now keeps only coverage-specific work:

- `make coverage`
- `make sanitizer-coverage-summary`
- coverage artifact upload
- sanitizer coverage artifact upload
- `make eval-local --output evaluation_runs/latest-ci-report.json || true`
- evaluation report artifact upload

Retrieval benchmark integrity artifact generation and upload moved to `benchmark-guard` to avoid duplicate execution.

## Snapshot Safety

`benchmark-guard` does not run `make retrieval-benchmark-baseline-refresh`.

The job is read-only for committed benchmark baseline snapshots:

- `docs/benchmarks/retrieval-benchmark-baseline.json`
- `docs/benchmarks/retrieval-benchmark-baseline.manifest.json`

The integrity steps may generate CI-only files under `docs/operations/artifacts/`, but the job must not refresh or rewrite committed baseline files.

## Legacy Guard Boundary

EVI-117 pytest coverage remains unchanged. The legacy `fixture_id` guard cases are still exercised by default tests and remain directly reproducible with:

```bash
uv run pytest -k fixture_id
```

The standalone job adds PR check visibility without changing retrieval ranking, metrics, fixture identity, or baseline refresh behavior.

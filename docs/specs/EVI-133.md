# EVI-133: Multi-tenant workspace/auth data model phase 1

This phase implements the first slice from `docs/design/multi-tenant-auth.md`:
durable workspace, user, membership, API key, and session tables without changing
existing tenant-owned application tables.

## Implemented

- Adds `workspaces`, `users`, `workspace_memberships`, `api_keys`, and
  `user_sessions` SQLAlchemy models and Alembic schema.
- Seeds a single local-development workspace during migration:
  `slug='default'`, id `00000000-0000-0000-0000-00000000defa`.
- Provides `ensure_default_workspace()` for create-all/local test paths that do
  not run Alembic.
- Provides `create_api_key()` and `verify_api_key()` with prefix lookup and
  HMAC-SHA-256 secret hashes. The raw issued key is returned once and is not
  persisted.
- Provides `create_user_session()` and `verify_session_token()` with
  HMAC-SHA-256 token hashes. Raw session tokens are not persisted; expired or
  revoked sessions fail verification.

## ACL boundary

Phase 1 intentionally does not change `src/ragrig/acl.py`. The auth layer only
reserves the mapping used by later request-context work:

- `users.id` maps to `user:<uuid>`.
- `workspace_memberships.group_ids` maps to `group:<group_id>`.
- Service API keys can set `principal_user_id` and `principal_group_ids` for
  stable service subjects.

Workspace filtering must happen before the existing ACL evaluator receives
principal subjects.

## Deferred workspace context gaps

This phase does not add `workspace_id` to existing tenant-owned tables and does
not change ingestion, retrieval, answer generation, vector sync, audit export,
backup/restore, task execution, or web-console list routes. Those paths still
operate in the legacy single-workspace mode and must be updated in a later phase
to require an explicit workspace context before tenant isolation is claimed.

Workspace CRUD endpoints and nullable `workspace_id` drafts on existing tables
are intentionally deferred to keep this migration non-invasive and preserve the
current local single-user experience.
